CSV Import Order Guide
======================

Document role
-------------
- This file only documents CSV import ordering and dependency rules.
- Feature behavior, authoring UX, and UE implementation roadmaps live in README.md, PROJECT_CONTEXT.md, and the UE5_Integration docs.

Why order matters
-----------------
- CSV import is "replace-all" per table.
- If referenced IDs are missing, import fails.
- If a CSV is partial, rows not included in that file are deleted for that table.
- Reset-based restore/rebuild operations preflight the complete CSV set before
  deleting the active SQLite data.
- A reset-based rebuild requires one source CSV for every persisted table,
  including empty tables.
- Factions must be imported before requirements and
  requirement_min_faction_reputation.
- Deleting or omitting a faction during a replace-all faction import
  cascade-deletes linked requirement_min_faction_reputation rows.

Current recommended order (for existing seed files)
---------------------------------------------------
1) stats
2) attributes
3) statuses
4) currencies
5) effects
6) abilities
7) ability_effect_links and ability_scaling_links
8) ability_relations
9) attribute_stat_links
10) characterclasses
11) items

The rebuild script (`scripts/rebuild_source_db.py`) follows this dependency
order for existing files and extends it for later world, narrative, economy,
and progression CSVs.

If/when you add NPC/enemy gameplay CSVs:
10) factions (if used by characters/rewards)
11) locations (if used by characters)
12) requirements (if used by travel gates, items, quests, events, encounters)
13) location_routes (after locations; after requirements if routes are gated)
14) travel_tuning (after locations/location_routes enum setup; no hard foreign keys)
15) characters
16) combat_profiles
17) interaction_profiles
18) items
19) item_stat_modifiers
20) item_attribute_modifiers
21) shops
22) shops_inventory
23) timelines
24) story_arcs
25) adventure_beats
26) lore_entries
27) quests
28) dialogues
29) dialogue_nodes
30) encounters
31) events
32) character_story_beats
33) adventure_beat_links (after adventure_beats and every linked target table)
34) location_pois (after locations, requirements, events, dialogues, encounters, items)
35) location_encounter_tables (after locations, requirements, encounters)
36) route_event_bindings (after location_routes, events, requirements)
37) location_creative_briefs (after locations)

General rule for new CSV files (going forward)
----------------------------------------------
1) Import independent "base tables" first (no foreign-key/reference dependencies).
2) Then import tables that reference only base tables.
3) Then import "composition/runtime" tables that reference many others.
4) Keep each imported CSV complete for its table (to avoid accidental row deletion).
5) Use source CSVs for SQLite regeneration. UE CSVs are generated DataTable exports.
6) Fix every preflight error before rebuilding; failed preflight leaves the
   active SQLite database untouched.

Quick dependency examples
-------------------------
- effects depends on: statuses, stats, attributes (optional), items (optional in related_items)
- items depends on: currencies (optional), effects (optional), stats/attributes (in modifiers)
- location_routes depends on: locations, requirements (optional)
- location_pois depends on: locations, requirements (optional), events (optional), dialogues (optional), encounters (optional), items (optional)
- location_encounter_tables depends on: locations, requirements (optional), encounters in encounter_entries
- route_event_bindings depends on: location_routes, events, requirements (optional)
- travel_tuning depends on: route type, place kind, and biome enum values only
- location_creative_briefs depends on: locations
- combat_profiles depends on: characters, stats, abilities, items/currencies/factions/quests (optional)
- ability_relations depends on: abilities
- ability_effect_links depends on: abilities and effects
- encounters depends on: characters (+ combat_profiles for Combat context)
- adventure_beats depends on: timelines/story_arcs and flags (optional)
- adventure_beat_links depends on: adventure_beats and the target selected by target_type
